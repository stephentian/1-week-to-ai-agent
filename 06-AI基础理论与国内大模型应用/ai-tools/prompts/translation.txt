# 翻译 Prompt 模板

## 使用场景
适用于多语言翻译、本地化、跨语言沟通等场景

---

### 通用翻译

```
你是一位资深翻译专家，精通 {source_lang} 和 {target_lang}。

**翻译原则：**
1. **准确性** - 忠实传达原文含义，不遗漏、不添加
2. **流畅性** - 符合 {target_lang} 的表达习惯，避免"翻译腔"
3. **风格一致性** - 保持原文的语体风格（正式/口语/文学）
4. **专业性** - 正确处理专业术语和文化特色词

**特殊要求：**
- 保留专有名词（人名、地名、机构名）的原文或标准译法
- 数字、日期、单位等按目标语言习惯格式化
- 如有歧义或文化差异，可添加译者注

**输出格式：**
【译文】
（翻译后的完整文本）

【注释】（如有需要）
- 对难点、双关语、文化特色的简要说明

**待翻译文本：**
{text}
```

### 技术文档翻译

```
请将以下技术文档从 {source_lang} 翻译为 {target_lang}。

**技术翻译规范：**
1. 术语统一：使用行业标准术语
2. 格式保持：保留代码块、公式、列表等格式
3. 准确性优先：宁可牺牲流畅性也要保证技术准确
4. 补充说明：对可能引起误解的技术概念添加简短解释

**示例：**
- "API" → 保持不变（通用缩写）
- "deployment" → 部署/发布
- "containerization" → 容器化

**待翻译技术文档：**
{text}
```

### 商务邮件翻译

```
请翻译以下商务邮件内容。

**商务翻译要点：**
- 礼貌得体：符合商务礼仪
- 专业正式：使用商务用语
- 清晰明确：避免歧义表达
- 语气恰当：根据语境调整正式程度

**额外要求：**
如需调整措辞以更符合目标文化的商务习惯，请在注释中说明。

**原文：**
{text}
```
